What is Distributed SCM?

The Distributed SCM has a source code placed in the Web where every user or developer can work on a given project/code without requiring them to share a common network. Each changes will be tracked separately and will have the unique id. It takes a peer-to-peer approach to version control. Distributed SCM synchronizes repositories by exchanging patches (set of changes) from peer to peer.


What is the benefit of the distributed SCM?


Some of the benefits are

It works offline.

All the activity can be done in the local system and only to push the changes developers needs to be online. If will not have any impact on your server downtime issue.

It‘s fast.

Diffs, commits and reverts are all done very fast than the centralized repository.

It handles changes well.

Distributed version control systems were built around sharing changes. Every change has a unique id which makes it easy to track.

Easy Branching and merging.

It has well organized branching strategy, because every developer “has their own branch”, every shared change is like reverse integration. But the unique id make it easy to automatically combine changes and avoid duplicates.

Artifactory FAQ

Artifactory is a binary repository manager. We use source control for our source code,
Artifactory is version control (and more) for your binary artifacts (jar/war files, etc).
Artifactory is also a place where you can put a shared library so that it is easily accessible in other projects across the enterprise.


Lights on Artifactory: https://www.youtube.com/watch?v=aa4YBDUDWy0

Table of Contents

  • What are the key scenarios to use Artifactory?
  • What is the maximum size of files uploaded?
  • What is the default repository size threshold?
  • How long will the uploaded files be retained?

What are the key scenarios to use Artifactory?

  • Rebuilding from source introduces points of failure and change.
    By having an artifact in a binary repository you have a versioned copy of the tested artifact.
  • Binaries are versioned; it is easy to have multiple versions of libraries across the projects.
  • Integrating with CI server like Jenkins will can have the clean builds which are automatically published with meta data about the build.
  • Artifactory has “virtual repositories” that are proxies to all of the “well known” maven repositories on the internet.
    This gives you access to most of the public shared libraries such as Apache Commons, Spring, Hibernate, etc..; through one conduit.

What is the maximum size of files uploaded?

By default, you can upload up to 100 MB at a time

What is the default repository size threshold?

By default, 20GB per project

How long will the uploaded files be retained?

By default, last 10 versions

SonarQube FAQ


Sonar is an open source platform used by development teams to manage source code quality. Sonar has been developed with a main objective in mind: make code quality management accessible to everyone with minimal effort

As such, Sonar provides code analyzers, reporting tools, defects hunting modules and Time Machine as core functionality. But it also embarks a plugin mechanism enabling the community to extend the functionality (more than 35 plugins available), making Sonar the one-stop-shop for source code quality by addressing not only developers but also manager’s needs.

In terms of languages, Sonar support analysis of Java in the core, but also of Flex (ActionScript 3), PHP, PL/SQL and Cobol through plugins (Open Source or commercial) as the reporting engine is language agnostic.


    Since version 2.0, Sonar enables to cover quality on 7 axes and so to report on:

  • Duplicated code
  • Coding standards
  • Unit tests
  • Complex code
  • Potential bugs
  • Comments
  • Design and architecture

Sonar can be used for one-off audits, but has been designed to support global continuous improvement strategy on code quality in a company and therefore can be used as a shared central repository for quality management.

Which programming languages will SonarQube support?

SonarQube supports C/C++, C#, Cobol, flex, java, JavaScript, objective-c, php, PL/I, PL/SQL, Swift, VB.net, VB6.0, Web (html, JSP/JSF), xml.

Can I add my own rules?

Yes, need to raise a request on tool once it’s up and running.

Which kind of software characteristics will SonarQube measure?

  1. Whether the coding has been done following a specific convention
  2. Whether well-known/established good practices have been followed and well-known/established bad practices have been avoided
  3. Are there any potential bugs and performance issues, security vulnerabilities?
  4. Is there any duplicate code?
  5. Is the code logic very complex?
  6. Whether the public API has good documentation and comments
  7. Whether the code has unit tests
  8. Whether the code follows good design and architecture principles

Where can I see SonarQube scan results?

By navigating from the you can see the list of results in the dashboard.

How can I access SonarQube?

You will be provided a separate project specific user credentials for SonarQube. You cannot use w3ID login for SonarQube

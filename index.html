About the Platform


Our vision is to facilitate the best in class instrumentation, leveraged by advanced software engineering methods to IBM GBS Projects across various Sectors and Service Lines, with an aim to standardize the Application Development practices accross GBS. The platform is built on IBM Cloud, adopts open source technologies, and promulgates advanced SE methods like IBM Bluemix garage Method, DevOps, Agile and Design Thinking.













Next Gen ADM Platform is self service enabled, and allows you to initiate creation of a toolchain for your projects within few clicks. You can choose the project profile that best suits your needs, Select the software engineering practices that are applicable for your delivery, and provision the toolschain that suits your needs. The platform tools are W3id authenticated, and employs IBM Bluegroups for project authorization. That means, you completely own the access control for the toolchain created by you.

What is Distributed SCM?

The Distributed SCM has a source code placed in the Web where every user or developer can work on a given project/code without requiring them to share a common network. Each changes will be tracked separately and will have the unique id. It takes a peer-to-peer approach to version control. Distributed SCM synchronizes repositories by exchanging patches (set of changes) from peer to peer.


What is the benefit of the distributed SCM?


Some of the benefits are

It works offline.

All the activity can be done in the local system and only to push the changes developers needs to be online. If will not have any impact on your server downtime issue.

It‘s fast.

Diffs, commits and reverts are all done very fast than the centralized repository.

It handles changes well.

Distributed version control systems were built around sharing changes. Every change has a unique id which makes it easy to track.

Easy Branching and merging.

It has well organized branching strategy, because every developer “has their own branch”, every shared change is like reverse integration. But the unique id make it easy to automatically combine changes and avoid duplicates.

  3. Under “Passwords and forms”, uncheck Enable Autofill to fill out web forms in a single click. To turn Autofill back on, simply check the box again.

Mozilla Firefox

  1. Click the menu button and choose “Preferences”.
  2. Select the “Privacy” panel.
  3. In the drop-down menu next to “Firefox will”; choose Use custom settings for history.
  4. Remove the check mark next to “Remember search and form history”.
  5. Close the “about: preferences” page. Any changes you’ve made will automatically be saved.

Safari

  1. Click on Safari in the program menu and select “Preferences” in the drop-down menu.
  2. Click the “Autofill” tab in the Preferences window.
  3. Click on each of the check boxes next to the autofill options to deselect them, turning off the autocomplete for each option.

How can I configure to trigger a build when code is checked in?

  1. Set the Jenkins job's Build trigger to Poll SCM, but do not specify a schedule
  2. Create a gitlab post-receive trigger to notify the URL
  3. Example http://yourserver/jenkins/git/notifyCommit?url=
  4. This will trigger all builds that poll the specified git repo.
  5. However, polling actually checks whether anything has been pushed to the used branch
